# STEP 5. 기능 명세서

**프로젝트명:** Feelio
**작성일:** 2026-07-03
**작성 기준:** 현재 코드 기준. "관련 API"는 백엔드가 아직 없으므로 ① 현재 동작하는 로컬 상태 함수(`src/stores/useFeelioStoreDc.js`)와 ② 향후 필요한 API(STEP 8에서 초안 정의 예정, 현재는 "추가 구현 필요")를 구분해 표기한다.

### FUNC-008 기록 등록 (지출/수입)

| 항목 | 내용 |
|---|---|
| 설명 | 사용자가 금액·감정·카테고리·상황·메모·일시를 입력해 지출 또는 수입을 등록한다 |
| 입력값 | type(expense\|income, 기본 expense), amount(숫자, 필수), emotion(8종 중 1, 필수), category(단일, 필수), situation(복수 선택, 선택), memo(선택), date(datetime-local) |
| 처리 과정 | ① 지출/수입 토글 → ② 금액 입력(숫자 외 문자 자동 제거, 천 단위 콤마 표시) → ③ 감정 말랑이 선택(선택 감정 색이 화면에 반영) → ④ 카테고리 칩 선택 → ⑤ 상황(복수)·메모·일시 입력 → ⑥ 저장 → (운영) 기록 저장 후 선택 상황들을 `transaction_situations`에 삽입(단일 트랜잭션) → ⑦ 토스트 "기록 저장됨", 폼 초기화, 홈 선택 날짜를 기록 날짜로 갱신 |
| 출력값 | 신규 거래 레코드 (emotion·category 단일, situations 배열, memo, occurredAt) |
| 예외 상황 | 금액·감정·카테고리 중 하나라도 없으면 저장 버튼 비활성 + "금액·감정·카테고리를 골라주세요" 안내. **확정 정책: 상황은 복수 선택 그대로 저장(N:M), 미선택 시 빈 배열. 메모 미입력 시 NULL 저장(기본 문자열 저장 안 함).** (현재 데모 코드는 상황 첫 번째 값만 저장·미입력 시 '기록'/'감정 기록' 문자열 저장 — 운영 전환 시 교체 대상) 일시 기본값 2026-07-01T21:30 고정도 현재 시각으로 교체 필요 |
| 관련 화면 | 지출·수입 기록 (`RecordPageDc.jsx`) |
| 관련 API | 현재: `addTransaction()` (로컬) / 향후: 거래 생성 API — 추가 구현 필요 |
| 구현 상태 | 구현 |
