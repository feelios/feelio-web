# Feelio — 우리 서비스는 무엇인가

> 작업 전 반드시 읽는다. 이 정의와 목적에 어긋나는 기능·문구를 만들지 않는다.
> 양쪽 repo(feelio-web, feelio-api)의 이 파일은 항상 동일하게 유지한다.

## 서비스 정의 (공식)

> **Feelio는 사용자가 수입·지출 내역을 기록하고 각 소비에 감정을 함께 남기면, 감정에 따른 소비 패턴 분석 인사이트를 제공하는 감정 소비 분석 서비스이다.**

- **Feel + I/O**: 감정을 입력(Input)하면, 소비 인사이트가 출력(Output)된다.

## 우리의 목적

1. **소비를 감정의 관점에서 돌아보게 한다.** 얼마를 썼는지보다 "어떤 마음으로 썼는지"를 남기게 한다.
2. **반복되는 감정-소비 패턴을 인사이트로 보여준다.** 예: "외로운 밤마다 배달 소비가 반복되고 있어요."
3. **수치로 평가·압박하지 않는다.** 무겁고 복잡한 회계 도구가 아니라, 가볍고 감성적인 기록·회고 경험을 제공한다. 사용자를 혼내는 서비스가 아니다.
4. **감정의 언어로 되돌려준다.** 텍스트·숫자 대신 감정 말랑이 캐릭터, 감정 색 캘린더, 감정 능선으로 보여준다.
5. **목표와 연결한다.** 감정소비를 줄이면 목표(평행우주의 다른 미래)에 가까워지는 흐름을 체험하게 한다.

## 핵심 사용자 루프

```text
기록 (금액 + 감정 8종 중 1 + 카테고리 + 상황·메모)
→ 홈 회고 (대표 감정 말랑이 · 감정 색 캘린더 · 월간 감정 능선)
→ 조회 (일별/월별/감정별 거래내역)
→ 인사이트 (AI 분석 · 평행우주 미래 비교)
→ 목표 (온보딩에서 설정, 소비 흐름과 연결)
```

## 감정 8종 (고정 — 이름 그대로 사용)

신남 · 설렘 · 뿌듯함 · 스트레스 · 외로움 · 화남 · 평온 · 무덤덤

## 표현 규칙

- **"가계부"라는 단어를 코드 주석·UI 문구·문서 어디에도 쓰지 않는다.**
  대체 표현: 감정 소비 분석 서비스, 감정 기반 소비 기록 서비스, 수입·지출 기록 서비스, 개인 소비 회고 서비스
- UI 문구는 부드러운 반말체를 유지한다. 예: "얼마나 썼어요?", "이 소비, 어떤 기분이었어요?", "기록 저장됨"
- 감정소비 누수율은 제거 확정된 개념이다 — 어떤 화면·API·문구에도 다시 만들지 않는다.

## 만들지 않는 것 (범위 제외)

- 이메일/비밀번호 회원가입·로그인 (소셜 로그인 전용: Google/Kakao/Naver)
- MyData·카드/계좌 자동 연동 (직접 기록 방식만)
- 커스텀 감정·태그 생성
- 친구 챌린지, 챗봇 상담, 관리자 기능
