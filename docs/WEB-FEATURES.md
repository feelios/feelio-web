## ⚠️ 프론트엔드 작업 핵심 원칙 (Core Directive)
**"현재 웹의 형태와 디자인(UI/UX)은 절대 훼손하지 않는다."**
모든 프론트엔드 작업은 기존 뷰(View) 템플릿과 스타일링을 100% 유지한 상태에서 **API 통신, 데이터 바인딩, 비즈니스 로직(오류 수정) 연결**에만 국한됩니다. 에이전트가 임의로 화면 디자인이나 레이아웃을 수정하는 것을 엄격히 금지합니다.

## 🚀 병렬 작업 규칙 (에이전트 간 충돌 방지)

* **도메인 단위 분할**: `F1` (인증/유저) 도메인과 `F2` (거래/메타) 도메인 등은 서로 다른 에이전트가 동시에 작업해도 무방합니다.
* **공통 컴포넌트 사전 고정**: 버튼, 모달, 입력창 등 공통 컴포넌트는 F1/F2 작업 전에 미리 확정해야 합니다. 에이전트가 임의로 `components/common/` 하위 코드를 수정하는 것을 금지합니다.
* **가장 우선되어야 할 작업**: `F1-2`의 **Axios Interceptor** 작업은 프로젝트 전체 API에 영향을 미치므로, 다른 API 연동 작업 이전에 **단일 에이전트가 가장 먼저 완수**해야 합니다.
* **캐시 키(Query Key) 엄수**: 표에 명시된 규칙(`['tx', 'list']` 등)을 절대적으로 따릅니다. 임의의 캐시 키를 생성하여 전역 상태가 꼬이는 것을 방지합니다.
* **브랜치명 일치**: 브랜치 충돌을 막기 위해 본 문서에 명시된 **브랜치명 그대로** 사용합니다. (1이슈 = 1브랜치 = 1PR)
