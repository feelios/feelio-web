# STEP 4. 요구사항 정의서

**프로젝트명:** Feelio
**작성일:** 2026-07-03
**작성 기준:** 현재 코드 기준 + STEP 1~3 승인 내용. 구현 여부는 프론트엔드 코드로 확인된 사실만 기재하고, 서버가 필요한 항목은 "미구현(백엔드 필요)"로 표기한다.

---

## 1. 프로젝트 개요

- **서비스 정의:** Feelio는 사용자가 수입·지출 내역을 기록하고 각 소비에 감정을 함께 남기면, **감정에 따른 소비 패턴 분석 인사이트**를 제공하는 감정 소비 분석 서비스이다. 사용자는 이를 통해 자신의 소비 흐름을 감정의 관점에서 돌아볼 수 있다.
- **목표:** 무겁고 복잡한 회계 도구가 아닌, 가볍고 감성적인 기록·회고 경험 위에 분석 인사이트를 얹는다.
- **범위:** 웹 애플리케이션 (데스크톱·모바일 반응형). 직접 기록 방식이며 금융기관 자동 연동은 범위에서 제외한다.
- **제거 확정 기능:** 감정소비 누수율(계산·표시)은 본 요구사항에 포함하지 않는다 (STEP 1·2 참조).

## 2. 사용자 유형

| 사용자 유형 | 설명 | 비고 |
|---|---|---|
| 일반 사용자 (회원) | 소셜 계정으로 로그인해 자신의 수입·지출과 감정을 기록하고 인사이트를 확인하는 사용자. 서비스의 유일한 주 사용자 유형 | 현재 코드 기준 단일 역할 |
| 비로그인 방문자 | 로그인 화면(서비스 소개)만 접근 가능 | 현재 코드 기준 |
| 관리자 | 별도 관리자 기능 없음 | 해당 없음 (필요 시 향후 검토) |
